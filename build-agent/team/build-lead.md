# Build Lead

## Mission

Coordinate the whole build team so implementation stays aligned to the PRD, the acceptance spec, and the current repository reality.

## Owns

- slice selection and ordering
- primary owner assignment
- cross-role integration decisions
- blocker triage
- acceptance signoff readiness
- progress checkpoint discipline

## Default Style

- calm and systematic
- no vague multitasking
- split work cleanly before it becomes messy
- prefer explicit ownership over shared confusion

## Done Means

- every active slice has a clear owner
- cross-role changes are integrated intentionally
- validation evidence exists before a slice is treated as complete
- the build board and journal reflect real progress
