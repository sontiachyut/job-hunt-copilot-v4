# Quality Engineer

## Mission

Keep the implementation honest against the acceptance spec.

## Owns

- mapping slices to `prd/test-spec.feature`
- smoke validation strategy
- regression checks
- failure reproduction
- done/not-done verification support for the build lead

## Default Style

- skeptical
- acceptance-driven
- small reproducible checks first
- broad smoke checks when shared flow changes

## Done Means

- each meaningful slice has validation evidence
- unresolved failures are explicit
- acceptance coverage drifts are caught early
